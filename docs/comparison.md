# Домовой vs существующие решения

Домовой не конкурирует с каждым инструментом в отдельности — он конкурирует с самой идеей держать десять разных приложений, данные которых не связаны между собой.

## Ключевые отличия

| | Домовой | Типичные альтернативы |
|--|---------|----------------------|
| Модель данных | Единый граф (`thing` + рёбра) | Отдельные базы в каждом приложении |
| Связи между доменами | Нативные рёбра | Ручное копирование / нет |
| Self-hosted | Да, обязательно | Редко или платно |
| AI-интеграция | Встроена (RAG, агенты, мульти-модель) | Плагины / нет |
| Автоматизация | Плейбуки (встроено) | Отдельный инструмент (n8n, Zapier) |
| Файловое хранилище | MinIO (S3, self-hosted) | Разные у каждого |
| Конфиденциальность | Все данные локально | Облако по умолчанию |
| Расширяемость | Воркеры, плейбуки, внешние модули | API (если есть) |

## По областям

### Задачи и GTD

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **Todoist** | Простота, кроссплатформенность | Нет зависимостей между задачами, нет связи с инвентарём | `depends_on`, `about`, `part_of` — нативно |
| **Things 3** | Отличный UX (только Apple) | Закрытая экосистема, нет автоматизации | Open, плейбуки |
| **OmniFocus** | Полноценный GTD, гибкие перспективы | Дорого, только Apple, нет AI | GTD через умные списки |
| **TickTick** | Удобный, дешёвый | Облако, нет графа связей | Self-hosted |
| **Linear** | Командная разработка | Не для личного/семейного | Не цель |

**Преимущество Домового:** задача может быть `about` сервера, `part_of` плейбука, `produces` документа — всё в одной модели без переключения инструментов.

### Управление знаниями и заметки

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **Notion** | Универсальность, шаблоны, командная работа | Производительность, слабый граф, облако | Граф нативный, self-hosted |
| **Obsidian** | Локальные файлы, граф ссылок, плагины | Нет баз данных, нет автоматизации | SurrealDB вместо Markdown-файлов |
| **Logseq** | Outliner + граф, открытый | Слабая производительность при росте | Масштабируется горизонтально |
| **Roam Research** | Bidirectional links, daily notes | Дорого, облако, закрытый | `references` ребро нативно |
| **Confluence** | Командная wiki, интеграции Atlassian | Тяжело, дорого, не для личного | Wiki-раздел в Домовом |

**Преимущество Домового:** заметка-wiki может ссылаться на реальный сервер, задачу или инцидент — не через текстовую ссылку, а через граф-ребро.

### Домашний инвентарь

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **Grocy** | Продукты питания, сроки годности, рецепты | Только еда, нет общего инвентаря | Общий граф с `needs`, `contains` |
| **Sortly** | QR-метки, фото, мобильный | Платно, облако, нет автоматизации | `identified_by` + vault |
| **HomeZada** | Дом целиком — ремонт, страховка | Американский рынок, облако | Аналогично, self-hosted |

**Преимущество Домового:** инвентарь связан с задачами (`needs`), финансами, местонахождением (`contains`) и историей перемещений — без дополнительных интеграций.

### Документы и файлы

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **Paperless-ngx** | OCR, теги, поиск по тексту, self-hosted | Только документы, нет связей с другими данными | `kind: файл` + `about` → любой узел |
| **Nextcloud** | Файлы, календарь, заметки, self-hosted | Разрозненные приложения внутри | MinIO + граф сверху |
| **Google Drive** | Удобство, совместная работа | Облако, нет графа | Self-hosted |

**Преимущество Домового:** документ `represents` физический оригинал, `about` медицинский приём, `can_access` конкретному члену семьи — граф знает контекст каждого файла.

### Личные финансы

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **YNAB** | Методология конвертов, обучение | Дорого, облако, только финансы | Периодические платежи в графе |
| **Firefly III** | Self-hosted, мощный, открытый | Нет связи с задачами/инвентарём | Платежи связаны с контекстом |
| **Actual Budget** | Локальный, быстрый, YNAB-методология | Только финансы | Встраивается в общую модель |

**Преимущество Домового:** платёж за страховку связан с конкретным имуществом, срок оплаты создаёт задачу автоматически через плейбук.

### IT-инфраструктура

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **NetBox** | DCIM/IPAM, стандарт индустрии | Только инфраструктура, нет задач/инцидентов в одной модели | IT — один из разделов общей схемы |
| **Netdata** | Мониторинг в реальном времени, агенты | Только метрики | Метрики через datasource (Prometheus) |
| **Grafana** | Дашборды, алерты, множество источников | Только визуализация | `kind: виджет` + datasource |
| **Ansible** | Автоматизация конфигурации | Отдельный инструмент | Плейбуки Домового + Ansible-воркер |
| **Home Assistant** | Умный дом, автоматизации | Только умный дом | Шлюз + плейбуки |

**Преимущество Домового:** сервер в графе — это узел, связанный с задачами обслуживания, терминальными сессиями, инцидентами и ответственным администратором одновременно.

### Автоматизация workflow

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **n8n** | Self-hosted, богатая библиотека нод, открытый | Отдельный инструмент, данные не в одной модели | Плейбуки встроены в граф |
| **Make (Integromat)** | Визуальный редактор, тысячи интеграций | Облако, дорого при росте | Self-hosted |
| **Zapier** | Простота, огромная экосистема | Дорого, облако, нет self-hosted | Self-hosted |
| **Temporal** | Durability, масштаб, enterprise | Сложность, только для разработчиков | Более простая модель |

**Преимущество Домового:** плейбук работает с теми же данными, что и остальная система. Результат шага — это узел в графе, а не JSON в сторонней БД.

### AI и чат

| Инструмент | Сильные стороны | Что не умеет | Домовой |
|-----------|----------------|-------------|---------|
| **ChatGPT** | Качество, экосистема | Облако, нет RAG из личных данных, нет истории в общем контексте | Все чаты в графе, RAG нативно |
| **OpenWebUI** | Self-hosted, Ollama, мульти-модель | Отдельный инструмент, нет интеграции с данными | Чаты — узлы в Домовом |
| **Lobe Chat** | Self-hosted, красивый UI, мульти-агент | Нет интеграции с личными данными | То же |
| **Jan** | Полностью локальный | Нет RAG из личных данных | То же |
| **Perplexity** | Поиск + AI | Облако, нет личного контекста | Self-hosted + личный граф |

**Преимущество Домового:** AI-чат знает твои задачи, инциденты, инвентарь — через `inputs` с SurrealQL-запросами без дополнительных интеграций.

## Когда Домовой — не лучший выбор

| Ситуация | Лучше использовать |
|----------|-------------------|
| Нужна командная работа с внешними людьми | Notion, Confluence, Linear |
| Простые задачи без связей | Todoist, Apple Reminders |
| Только финансы и бюджет | YNAB, Actual Budget |
| Только мониторинг инфраструктуры | Grafana + Prometheus |
| Нет желания администрировать self-hosted | Любой SaaS |
| Нужна корпоративная поддержка | Enterprise-инструменты |

---

## Итог

Домовой — это не «ещё один менеджер задач». Это инфраструктура для личных и семейных данных, где граф связей между доменами важнее, чем функциональность каждого домена по отдельности.

Ценность проявляется не сразу, а когда задача об автомобиле связана с расходами на него, напоминает о ТО, хранит документы и историю ремонтов — и всё это видно из одного места без копирования данных между приложениями.
