# Домовой

Семейная система управления жизнью на основе графа. Заменяет разрозненные приложения — списки покупок, трекер задач, домашний инвентарь, медкарту, юридические дела, долги, доставки, обучение — единой моделью данных.

## Идея

Всё есть **вещь**. Полка, мотоцикл, задача, рецепт, врач, жалоба, трекинг-номер, вопрос из квиза — один тип узла. Смысл задаётся рёбрами между узлами.

```
Задача: ТО мотоцикла
  ├── assigned_to → Папа
  ├── part_of     → Подготовка к сезону
  ├── depends_on  → Купить масло и фильтры
  └── produces    → Запись о ТО (исторический факт)
```

Это позволяет моделировать что угодно без новых таблиц: периодические платежи, цепочки юридических обжалований, многоперевозчиковые доставки, адаптивные квизы, контекстные GTD-задачи.

## Что умеет

| Область | Возможности |
|---------|-------------|
| **Инвентарь** | Где хранится, кому одолжено, когда куплено, сколько осталось |
| **Задачи** | Подзадачи, зависимости, блокирующий каскад, повторение, расписание |
| **Семья** | Совместные и личные задачи, временные группы, обещания |
| **Медицина** | Приёмы, назначения, направления, цепочка ожидания записи |
| **Юриспруденция** | Обращения, обжалования, fan-out по инстанциям |
| **Платежи** | Периодические шаблоны, история оплат, просрочки |
| **Доставки** | Сложный трекинг, смена перевозчика, разбитые посылки |
| **Обучение** | Курсы, квизы, попытки, AI-адаптация по слабым темам |
| **Документы** | Физические оригиналы + цифровые копии, доступ без глобальных прав |
| **GTD** | Контекстные задачи, inbox, энергия задачи, сезонные паттерны |

## Граф: 1 тип узла, 21 ребро

```
contains      — где физически находится
part_of       — часть чего / подзадача / запуск шаблона
assigned_to   — кто отвечает
depends_on    — задача ждёт другой
about         — касается этой вещи или решения
filed_with    — подано в инстанцию
needs         — нужно купить (quantity, unit)
requires      — плановый расход шаблона
produces      — результат выполнения
used          — фактический расход
participant   — участник с ролью
located_at    — где происходит событие
lent_to       — отдано во временное пользование (since)
borrowed_from — взято у человека (since)
triggered_by  — активируется при встрече с X
expert_in     — эксперт в области
answered      — попытка → вопрос (chosen, correct, points_earned)
promised_to   — кому дано обещание
represents    — цифровая копия → физический оригинал
can_access    — кто имеет доступ
related_to    — произвольная связь (label)
```
